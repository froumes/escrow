Config-first setup

Most serious users want the knobs up front.

The config section now explains what matters instead of just dumping a tiny code block in the middle of the page.

Account and module control

Switch Auction House and Bazaar modules independently and define one or more usernames.

Speed tuning

Adjust command pacing, bed interaction timing, and other latency-sensitive values directly in TOML.

Web and Discord integration

Set the web panel port and webhook destination without touching application code.

config.toml Starting point 
# Minecraft account(s)
ingame_name = "YourUsername"

# Flip modules
enable_ah_flips = true
enable_bazaar_flips = true

# Timing and speed
command_delay_ms = 500
bed_spam_click_delay = 100
bed_pre_click_ms = 30

# Web panel
web_gui_port = 8080

# Notifications
webhook_url = "https://discord.com/api/webhooks/..."
Install

A first run path that does not make people dig through the repo.

The install section now keeps the Linux VPS flow obvious, tabbed, and copyable.

Step 1

Download a release or loader

Pick the binary path if you want the direct executable, or the loader if you want the latest release fetched automatically.

Step 2

Run the binary

Launch it, complete the Microsoft login flow, and let the app create the local config file.

Step 3

Tune config and open the panel

Adjust timing, modules, usernames, and web settings before you settle into longer sessions.

bash - Linux VPS

$ wget https://github.com/froumes/escrow/releases/latest/download/twm-linux-x86_64

$ chmod +x twm-linux-x86_64

$ ./twm-linux-x86_64

bash - Linux VPS loader

$ wget https://github.com/froumes/escrow/releases/latest/download/TWM-loader-linux-x86_64

$ chmod +x TWM-loader-linux-x86_64

$ ./TWM-loader-linux-x86_64

The loader path is useful if you want it to pull the latest published build automatically.
